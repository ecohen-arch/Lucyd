# Lucyd Help Center Architecture
## Intercom to Gorgias Migration Plan

---

## Executive Summary

This plan provides a complete Help Center architecture for lucyd.co, designed to reduce ticket volume by 30-50% through self-service deflection while integrating seamlessly with Gorgias macros, rules, and automations.

**Key Goals:**
- Reduce support ticket volume
- Improve self-service resolution rates
- Align Help Center with Gorgias automation capabilities
- Create scalable knowledge base for smart eyewear + prescription products

## 3. User Flows for Main Scenarios

### Flow 1: "Where Is My Order?" (WISMO)

```
[Chat Widget] -> "Track My Order" button
       |
       v
[Auto-prompt] -> "Enter order number or email"
       |
       v
[Gorgias pulls Shopify data]
       |
+------------------+-------------------+------------------+
| Status: Shipped  | Status: Preparing | Not Found        |
+------------------+-------------------+------------------+
| Show tracking    | Show processing   | Ask to verify    |
| URL + delivery   | timeline +        | details + offer  |
| estimate         | Rx explanation    | agent help       |
+------------------+-------------------+------------------+
       |
       v
[Resolved?] -> Yes: Close (no ticket) | No: Create ticket with context
```

**Gorgias Config:**
- Rule trigger: "where is", "track", "order status", "shipping"
- Auto-tag: `ORDER`, `wismo`
- Macro: `ORDER-WISMO-Auto`

---

### Flow 2: "Prescription Didn't Upload"

```
[Chat Widget] -> "Prescription Help" -> "Can't upload"
       |
       v
[Auto-response with troubleshooting]:
- Accepted formats: JPG, PNG, PDF (<5MB)
- Quick fixes: Clear photo, different browser, clear cache
       |
       v
[Resolved?]
       |
       v
Yes: Close | No: Offer to accept Rx via email reply
       |
       v
[Create ticket] -> Agent processes manually
```

**Gorgias Config:**
- Rule trigger: "can't upload", "prescription upload", "upload failed"
- Auto-tag: `PRESCRIPTION`, `upload-issue`
- Macro: `RX-Upload-Help`

---

### Flow 3: "Product Not Connecting"

```
[Chat Widget] -> "Technical Support" -> "Bluetooth pairing"
       |
       v
[Step 1] -> "Which device?" -> iPhone / Android / Other
       |
       v
[Step 2] -> Device-specific pairing instructions
       |
       v
[Resolved?] -> No -> [Step 3] Full reset instructions
       |
       v
[Resolved?] -> No -> Collect device model + OS version
       |
       v
[Create ticket with diagnostic info]
```

**Gorgias Config:**
- Rule trigger: "bluetooth", "won't connect", "pairing", "disconnect"
- Auto-tag: `TECHNICAL`, `bluetooth`
- Macro: `BT-Troubleshooting-Flow`

---

### Flow 4: "Refund Status"

```
[Chat Widget] -> "Returns & Refunds" -> "Check refund status"
       |
       v
[Auto-prompt] -> "Enter order number"
       |
       v
[Gorgias checks Shopify refund data]
       |
+-------------------+--------------------+------------------+
| Refund Processed  | Return Received    | No Return Found  |
+-------------------+--------------------+------------------+
| Show amount +     | Show "processing"  | Show return      |
| timeline by       | + 3-5 day          | eligibility +    |
| payment method    | estimate           | offer to start   |
+-------------------+--------------------+------------------+
```

**Gorgias Config:**
- Rule trigger: "refund", "return status", "money back"
- Auto-tag: `RETURN`, `refund-status`
- Macro: `RETURN-Refund-Status`
